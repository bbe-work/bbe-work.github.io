I en prioritetskö (priority queue) spelar insättningsordningen ingen roll, men det som är garanterat är att det objekt osm har högst prioritet av de som finns i kön är det element som kommer att lämna kön vid nästa borttagning. En liknelse från verkligheten som ligger närmast är en sk VIP-kö. Den som är *viktigast*/*har högst status* är den som lämnar kön först oavsett när personen i fråga kom till kön.

Prioritetsköns beteende, dvs uppsättning operationer, motsvaras av

- enqueue - sätt in ett element i kön
- dequeue - ta bort det element som har högst prioritet
- peek - "kika" på det noden som har högst prioritet utan att ta bort det
- size - avgör storleken, dvs antalet element, i kön

Det finns dessutom ett krav på elementen som ska placeras i prioritetskön och det är att det ska vara möjligt att jämföra elementen. För att uppnå kravet på elementen använder vi det generiska interfacet (som vi har tittat på tidigare)

```csharp
public interface IComparable<in T>
{
    int CompareTo(T? other);
}
```

Metoden `int CompareTo(T? other)` returnera
- *< 0* om anropande objekt (this) är *mindre än* other
- *0* om anropande objekt (this) är *lika med/ekvivalent* other
- *> 0* om anropande objekt (this) är *större än* other

## Interface för Prioritetskö

Vi skapar ett generiskt interface med krav på att den generiska datatypen ska vara jämförbar.

```csharp
interface IPriorityQueue<T> where T : IComparable<T>
{
    void Enqueue(T elem);
    T Dequeue();
    T Peek();
    int Size();
}
```
Nu har vi sett till att det finns krav på metoden `CompareTo` för den generiska datatypen som ska hanteras i prioritetskön.

### Använda ADT Prioritetskö

Anta att interfacet `IPriorityQueue` har implementerats av den generiska klassen `PriorityQueue`. 

Vad förväntar vi oss av klassen PriorityQueue? 

- Att vi kan skapa en tom prioritetskö
- Att vi kan placera element i prioritetskön.
- Att vi kan ta bort element noden som har högst prioritet.
- Att vi kan kika på det element som har högst prioritet utan att ta bort det    

Vi exemplifiera med int som vi vet uppfyller kravet på den generiska datatypen, nämligen att `CompareTo` finns.
    
```csharp
IPriorityQueue<int> pq = new PriorityQueue<int>();

pq.Enqueue(222); // 222 är placerad i prioritetskön
pq.Enqueue(111); // 111 är placerad i prioritetskön
pq.Enqueue(444); // 444 är är placerad i prioritetskön
pq.Enqueue(333); // 333 är är placerad i prioritetskön

int res = pq.Dequeue(); // 444 har högst prioritet och tas bort från prioritetskön ...

res = pq.Peek(); // 333 har högst prioritet och returneras till res men tas inte bort
res = pq.Dequeue(); // 333 har högst prioritet och tas bort från prioritetskön ...
res = pq.Dequeue(); // 222  har högst prioritet och tas bort från prioritetskön ...

```

Vi exemplifiera även med Car (enligt ovan) som också uppfyller kravet på `CompareTo` 
    
```csharp
IPriorityQueue<Car> pqCars = new PriorityQueue<Car>();

pqCars.Enqueue(new Car("Lexus", 2019)); 
pqCars.Enqueue(new Car("BMW", 2015)); 
pqCars.Enqueue(new Car("Volvo", 2023)); 
pqCars.Enqueue(new Car("Nissan", 2007)); 

Car res = pqCars.Dequeue(); // bilobjektet motsvarande Volvon av årsmodell 2023 tas bort

res = pqCars.Peek(); // bilobjektet motsvarande Lexus av årsmodell 2019 returneras men tas inte bort
res = pqCars.Dequeue(); // bilobjektet motsvarande Lexus av årsmodell 2019 tas bort
res = pqCars.Dequeue(); // bilobjektet motsvarande BMW av årsmodell 2015 tas bort

```

## Sammanfattning

- ADT Prioritetskö garanterar att det element som har högst prioritet kommer att vara det element som lämnar kön.
- Beteendet, dvs de operationer som ADT:n Prioritetskö har är
    - enqueue - sätt in ett element i kön
    - dequeue - ta bort det element i kön som har högst prioritet
    - peek - "kika" på det element i kön som har högst prioritet utan att ta bort det
    - size - leverera hur många element prioritetskön innehåller
