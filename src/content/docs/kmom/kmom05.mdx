I detta kursmoment ska vi bekanta oss med fler ADT:er som tillhör kategorin **Träd**. Vi kommer att gå igenom vad ett träd är och därefter fördjupa oss **binära träd** och specifikt Binära sökträd, sk. **BST** varav **AVL-träd** ett sådant.
Slutligen går vi igenom en anna typ av binärt träd som kallas **Heap**.

### Artiklar

Jobba igenom följande artiklar.

1. Vi börjar med att belysa begreppet [Träd](../../laromaterial/kmom05/trad).

2. Det finns olika typer av träd där binära sökträd ett det sådant. Det går vi igenom här [BST](../../laromaterial/kmom05/bst).

3. Vi fortsätter med att få en inblick i ett självbalanserat träd som kallas [AVL](../../laromaterial/kmom05/avl).

4. Avslutningsvis kikar vi på ett träd som kallas [Heap](../../laromaterial/kmom05/heap)

5. Använd ytterligare källor om du behöver så att du känner att du fått förståelse och kunskap om de koncept och begrepp som ingår i kmom:et.

## Målet med kmom05

Efter att ha jobbat igenom kmom05 så ska du:

- förstå begrepp och koncept som ingår i Träd
- kunna beskriva principen för BST och AVL
- kunna redogöra för de operationer som finns för BST
- kunna implementera BST med länkning som intern datastruktur
- kunna beskriva och redogöra för trädet Heap
- kunna implementera Heap med array som intern datastruktur 
- kunna redogöra för tidskomplexiteten för respektive operation för BST och Heap
